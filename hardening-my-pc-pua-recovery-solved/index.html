Based on the issues described with your HP EliteDesk 800, here is a structured approach to troubleshooting the Wi-Fi, LAN/WLAN switching, and fan speed behavior.

1. Fixing the Wi-Fi/Bluetooth Driver Issues

If the driver is installed but not loading (often appearing as Code 10 or Code 43 in Device Manager), the card might be stuck in a low-power state or experiencing a hardware-level conflict.

Perform a “Cold” Boot: Shut down the computer, unplug the power cable, hold the power button for 30 seconds to drain residual power, plug it back in, and turn it on.

Disable Fast Startup in Windows: This often prevents Wi-Fi drivers from loading correctly on HP EliteDesks. Go to Control Panel > Power Options > “Choose what the

 

power buttons do,” click “Change settings that are currently unavailable,” and uncheck “Turn on fast startup.”

Remove “HP LAN/WLAN/WWAN Switching UWP Service”: This service can cause conflicts, forcing the Wi-Fi off even if it is enabled in BIOS. Go to Services in Windows, find this service, stop it, and set it to Manual or Disabled.

Check BIOS “Device Options”: Ensure that Embedded WLAN Device and Bluetooth are actually enabled in the BIOS under Advanced > Built-in Device Options.

In the HP EliteDesk 800 SFF BIOS, the Allow OPAL Hard Drive SID Authentication setting controls whether third-party software is allowed to take ownership of a Self-Encrypting Drive (SED).

 

HP Support Community

What it does

 

Ownership Control: When unchecked (the default), the BIOS blocks attempts to authenticate the SID (Security ID) authority. This prevents unauthorized software or malicious actors from taking ownership of a drive and setting up their own encryption keys.

Provisioning Permission: Checking this box allows an application (like Samsung Magician, BitLocker with Hardware Encryption, or enterprise tools like Trellix) to “pair” with the drive, set a password, and manage the hardware-level encryption.

 

HP Support Community

+3

If you are NOT using OPAL applications

If you aren’t using specific hardware-encryption management software, this setting has no functional impact on your daily use.

 

Leave it Unchecked: For most users, keeping this disabled is the standard security recommendation. It acts as a “freeze” command that prevents any software from accidentally or maliciously locking your drive at the hardware level.

Data Security: Your data is still accessible and operates normally. Standard software encryption (like default BitLocker) does not require this setting to be enabled because it handles encryption at the OS level, not the drive’s firmware level.

The “Block SID” concept: In many BIOS versions, this is essentially the “Disable Block SID” toggle. “Blocking SID” is a security feature that stops the drive from being provisioned; “Allowing Authentication” unblocks it so you can set it up.

You should only enable this if you specifically intend to:

Enable Hardware-Accelerated BitLocker (where the SSD does the work instead of your CPU).

Use enterprise security suites like McAfee/Trellix Drive Encryption or ESET Full Disk Encryption.

Manually manage a Self-Encrypting Drive (SED) using tools like sedutil.

1. Why you should leave it Disabled

Preventing “Hardware Hijacking”: Enabling this setting allows software to take low-

 

level ownership of your drive’s firmware. Some advanced malware or unwanted security tools could theoretically use this to “lock” your drive or create a persistent hidden partition that survives a Windows reinstall.

Not Needed for BitLocker: Windows BitLocker uses your CPU to handle encryption by default (Software Encryption). It does not need the OPAL SID setting to function.

Security Risk: Leaving it disabled acts as a “freeze” on the drive’s internal security settings, preventing any app from changing how the hardware itself is locked or unlocked.

In the BIOS of an HP EliteDesk 800 (G1 through G5 models), the Absolute Persistence Module (or its older name, Computrace) is found under the Security

 

tab.

Exact Location in BIOS

Restart your computer and tap F10 repeatedly to enter the BIOS Setup.

Navigate to the Security tab using the arrow keys.

Scroll down to find Absolute Persistence Module or Computrace.

On some older versions (like G1/G2), it may be listed under a sub-menu called Anti-Theft or System Security.

In newer revisions, it is usually located near the bottom of the main Security list.

Important Settings to Know

Disabled/Deactivated: The feature is off but can be turned back on later if needed.

Permanently Disable: This is a one-way action. Once you select this and save, the module is hardware-locked and can never be re-enabled on that motherboard.

Greyed Out: If the setting is greyed out and

 

says “Active,” the module is already linked to a subscription and cannot be changed from the BIOS. You would need to contact Absolute Software Support to release it.

To further harden your HP EliteDesk 800 against malware and Potentially Unwanted Applications (PUA) at the BIOS level, you should focus on enabling “self-healing” and “isolation” features that HP builds into its business-class machines.

1. Enable HP Sure Start

This is your most powerful tool. It uses a dedicated security chip to verify your BIOS is clean every time you boot.

Location: BIOS > Security > Sure Start Configuration.

Action: Ensure HP Sure Start is Enabled.

 

Why: If malware tries to infect your BIOS (the “firmware”), Sure Start will detect the change, delete the infected version, and replace it with a clean “golden copy” automatically.

 

HP

+4

2. Set a BIOS Administrator Password

Without this, any malware that gains “Admin” rights in Windows can potentially change your BIOS settings from inside the operating system.

Location: BIOS > Security > Set BIOS Administrator Password.

Action: Create a strong password.

Why: This locks your BIOS settings. Even if a PUA tries to disable Secure Boot or enable Absolute Persistence, it will be blocked because it doesn’t have your BIOS password.

 

 

HP Wolf Security

+4

3. Enable DMA Protection (Direct Memory Access)

This prevents “DMA attacks” where a malicious device plugged into a port (like a rogue USB or Thunderbolt device) tries to read your computer’s memory directly.

Location: BIOS > Advanced > System Options (or Security > DMA Protection).

Action: Enable Pre-boot DMA Protection.

Why: It creates a barrier that prevents external hardware from bypassing Windows security to steal data or inject code.

 

HP Wolf Security

+2

4. Enable Virtualization-Based BIOS Protection

 

Location: BIOS > Advanced > System Options.

Action: Enable Virtualization Technology (VTx) and VT-d.

Why: Modern Windows security features (like Core Isolation and Memory Integrity) require these BIOS settings to be on. They create a “bubble” around your sensitive system processes so malware can’t touch them.

The device manager shows it on an active and fine it just does not show in the settings like the radio might be off and I’ve already went through the steps in services to make sure all the according services that are affected the Wi-Fi to be on

If Device Manager says the card is “working properly” but Windows says the radio is off, you’re likely stuck in a “Hardware Radio Kill” state. On HP EliteDesks, this is almost always caused by a conflict between the BIOS and Windows power management.

Here is the “last mile” fix for this specific scenario:

 

1. The “Power Management” Handshake Fix

Windows might have “parked” the radio to save power, and now it can’t wake it back up.

In Device Manager, right-click your Intel Wi-Fi adapter > Properties.

Go to the Power Management tab.

Uncheck “Allow the computer to turn off this device to save power.”

Click OK, then Disable the device in Device Manager, wait 5 seconds, and Enable it.

2. Check for “HP Hotkey Support”

HP EliteDesks use a specific driver called HP Hotkey Support (or HP Collaboration Keys). If this driver is corrupted or missing, Windows might think the “Physical Wireless Switch” is in the OFF position, even if your model doesn’t have a physical button.

Check your Apps & Features list for “HP

 

Hotkey Support.” If it’s there, try uninstalling it. Windows 11 often manages the keys better without the old HP software.

3. Clear the CMOS (The “Nuclear” Option)

Sometimes the BIOS “Hardware Radio” flag gets stuck in an “Off” state that even a BIOS update won’t fix.

Unplug the PC.

Open the case and look for a small yellow button on the motherboard labeled CMOS.

Hold it down for 5-10 seconds.

Plug back in and boot up. This forces the BIOS to re-detect the Wi-Fi card’s radio state from scratch.

4. Use the “Radio Enable” Command

Open PowerShell (Admin) and paste this command to see if Windows thinks the radio is software-locked:

Get-NetAdapterWirelessProperty

Look at the Status or State. If it says

 

“Hardware Off” or “Software Off,” try this to force it on:

netsh interface set interface “Wi-Fi” admin=enabled

(Replace “Wi-Fi” with whatever the name is in your Network Connections).

One quick question: On your keyboard, does your F12 key have a small light on it (usually an orange or white LED)? If that light is orange, the hardware radio is locked by the BIOS.
